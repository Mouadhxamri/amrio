# amrio development rules

- This project is amrio.
- amrio is an AI workspace for customer support teams.
- The product helps support teams answer tickets faster using internal knowledge, shared context, and workflow automation.
- Use Next.js App Router, TypeScript, Tailwind, Supabase, and Vercel.
- Use Supabase SSR patterns for auth.
- Prefer simple, typed solutions.
- Do not add dependencies unless necessary.
- Do not change architecture without asking first.
- Implement one task at a time.
- Summarize all changed files before stopping.
- Ask before making destructive changes.
- Plan first, code second.
- Do not refer to competitors in product copy, legacy files, or internal planning docs unless explicitly asked for competitive analysis.
